# 司法院裁判書閱讀助手

瀏覽司法院法學資料檢索系統時，於頁面側緣注入「判決架構」導覽與「參照」引文清單；複製文字時自動移除換行、附上裁判字號；並提供瀏覽器側邊欄「判決剪貼簿」，以卡片形式保存當次瀏覽期間複製之段落，便於跨分頁貼至 Word、Google Docs、Obsidian 等外部編輯器。

- **嚴格區分援引與反駁**：行內式引用須於尾端出現強結論語（意旨參照／可參／可資參照／足資參照等）始接受，避免將法院正在排斥之先例誤標為參照。

- **後台新增「參照耳標」開關**（預設開）。

歷史版本變更紀錄（v0.2.5 以前）請參閱 [STORE_LISTING.md](./STORE_LISTING.md) 之「更新」章節。

## ✨ 功能

1. **判決架構側欄**（滑鼠停駐式頁籤）
   - 偵測台灣判決常見之完整 6 層編號階層：`壹／貳` → `一／二` → `(一)(二)／㈠㈡` → `1.／⒈／１．` → `(1)／⑴／（１）` → `①／②／③`
   - 全形、半形、圓圈與帶括號之阿拉伯數字一律歸為同層，視覺縮排一致
   - 另偵測段落標題：`主文／事實／理由／事實及理由`（後者常見於地院刑事判決）
   - 判決末尾「附表」章段亦納入耳標，支援 `附表`、`附表一／二`、`附表甲／乙`、`附表壹／貳`、`附表1／2` 等編號形式
   - 「主文」後接無編號正文者，自動補入一筆子條目供跳轉
   - 點擊條目平滑捲動至對應段落，並自動扣除固定表頭高度
   - 相容「判決易讀小幫手」所包裹之 `<a>` 專有名詞連結，不致截斷標籤文字
   - 自動略過引用條文：`「按 X 法第 N 條規定：⋯」` 內之 `一、二、三` 不會被誤納入耳標
   - 卡片寬度依當下展開之最深層級**動態調整**，淺層判決保持精簡，深層判決自動加寬以免標籤被截斷
   - 耳標可於設定頁切換停靠於頁面**左側或右側**

2. **參照頁籤（裁判意旨引文清單）**（與判決架構共用側邊欄之獨立頁籤）
   - 抽取正文所有權威意旨引用，依原文出現順序條列於側邊欄卡片
   - **八類引用完整涵蓋**：最高法院判決／裁定（`最高法院112年度台上字第187號`）、最高法院大法庭裁定（`最高法院刑事大法庭109年度台上大字第1869號`）、最高法院民刑事庭會議決議（`最高法院94年度第6、7次刑事庭會議決議`，含多次編號）、法律座談會決議／審查意見（`本院暨所屬法院101年法律座談會民執類提案第18號決議`）、大法官釋字解釋（`司法院釋字第679號解釋`）、憲法法庭判決（`憲法法庭113年憲判字第3號判決`）、最高行政法院判決／裁定（`最高行政法院110年度判字第123號判決`）、最高行政法院庭長法官聯席會議決議（`最高行政法院98年度6月份第1次庭長法官聯席會議決議`）
   - **同一括號內多案號拆分**：`（最高法院112年度台上字第187號、113年度台上字第4204號）` 類並列寫法可逐一解析，第二筆以後省略「最高法院」前綴亦能正確辨識；跨類混列（如判決與大法庭併列）亦可分別歸類
   - **兩種引用格式相容**：除括號式 `（最高法院…）`，亦支援行內無括號寫法（如 `有最高法院X年度台上字第N號、第M號判決要旨，及同院Y年度第K次民事庭決議可參`），以鄰近分群及尾隨結論語擴張辨識
   - **同段多引用接續**：同段內多筆引用時，第二、三筆之跳轉範圍自前一筆結束處起算，不再重複覆蓋段首共同前言；卡片顯示前 90 字摘要
   - **八類案號以色系分類**：最高法院判決（主題色）／大法庭（暖琥珀）／庭會議（中性灰）／法律座談會（藕紫）／釋字解釋（深玫瑰紅）／憲法法庭（淺玫瑰紅）／最高行政法院判決（青綠深）／最高行政法院聯席會議（青綠淡）
   - **點擊切換暫態高亮**：點擊條目後以鵝黃暫態高亮整段意旨，同條再點擊即清除高亮
   - **一鍵複製並存入卡片**：每筆條目右側提供直向「複製」按鈕，點擊即將該筆意旨段落正規化後寫入系統剪貼簿（是否附上本篇裁判字號依後台設定），並同步推入判決剪貼簿卡片清單；同時插入書籤錨點供日後「前往」跳轉
   - 正文中之 `（…）` 引文附記以柔和天藍灰底**長駐高亮**，閱讀時可快速掃視
   - 若頁面未偵測到任何權威意旨引用，參照頁籤不顯示，以保持側邊欄精簡

3. **智慧複製**
   - 選取文字後複製，自動移除所有換行與中日韓字元間之多餘空白
   - 保留半形英數字之間的空格（例如 `NT 300` 不會變為 `NT300`）
   - 尾端自動附上 `（<裁判字號>意旨參照）` —— 字號自頁面「裁判字號：」欄位擷取，可於設定頁或側邊欄一鍵關閉
   - **Cmd+X／Ctrl+X 快速鍵**：同樣正規化後寫入剪貼簿，惟**不**存入剪貼簿卡片；適用於僅需一次性貼至外部工具、不欲累積卡片之情境。可編輯欄位（搜尋框、登入框等）保留原生剪下行為，不受攔截
   - Windows 與 macOS 共用同一套複製處理流程

4. **判決剪貼簿側邊欄（瀏覽器原生側邊欄）**
   - 每次 Cmd+C／Ctrl+C 複製後，自動將「去換行＋附字號」之完整文字推入瀏覽器原生側邊欄（Chrome Side Panel），以卡片形式保存當次瀏覽期間之所有判決段落
   - 側邊欄由點擊工具列上之擴充功能圖示啟動，**不限於裁判書頁面**——切至 Google Docs、Word Online、Obsidian Web、Notion 等任何分頁皆可檢視同一份卡片清單；點擊卡片上之「複製」鈕即可貼上
   - 卡片內容包含：來源標籤（裁判書／判解函釋／內網判解函釋／內網裁判書）、裁判字號（可點擊於新分頁開啟原判決頁）、複製時間、完整內文；色系依來源系統自動切換，採用司法院官方色票 `#336633`／`#336666`／`#006699`
   - **前往原文段落**：每張卡片之「前往」按鈕可直接跳回該段落於原判決頁之位置，並以黃色高亮標示。複製當下即於選取文字之起迄位置插入隱形錨點；「前往」以 DOM 直接定位，不倚賴網址比對。原分頁若仍開啟即切換過去；已關閉或已導覽至其他判決者則自動開啟新分頁載入原文再跳轉
   - **#標籤**：每張卡片可加入自訂 #標籤，以彩色膠囊樣式顯示，色彩依來源主題；搜尋列下方自動彙整所有標籤，點擊即可快速篩選
   - **備註**：每張卡片可撰寫自由文字備註，儲存後直接顯示於卡片上；點擊備註預覽可再次編輯
   - **關鍵字搜尋**：側邊欄頂端提供搜尋列，可即時篩選卡片內文、字號、備註或 #標籤
   - **拖放排序**：按住卡片拖曳至目標位置即可重排，便於依書狀結構整理引文
   - **附字號即時切換**：側邊欄頂端之「附上字號 開／關」切換與設定頁之「複製時自動附上裁判字號」共用同一設定，變更後即時同步至所有分頁與設定頁，無須開啟後台
   - 完全重複之內容自動去重，並以浮動提示顯示「已複製過相同內容」
   - 字體大小可四段調整（小／中／大／特大），標籤與字號隨內文同步縮放，偏好跨瀏覽期間保留
   - 支援匯出為 `.txt` 純文字或 `.md` Markdown 檔（專為 Obsidian 設計之區塊引用格式），匯出時一併攜帶 #標籤與備註
   - **僅保留當次瀏覽器開啟期間之紀錄**：資料僅存於 `chrome.storage.session`（記憶體），關閉瀏覽器即自動清空；不寫入硬碟、不同步、不傳送至任何外部服務

5. **自動主題配色**
   - `legal.judicial.gov.tw`（FINT 法令判解系統） → 深青主色 `#336666`
   - `judgment.judicial.gov.tw`（FJUD 裁判書系統） → 深綠主色 `#336633`
   - `legal.law.intraj`／`judgment.law.intraj`（司法院內網） → 海軍藍主色 `#006699`，與內網頁面背景相融
   - 每次內嵌框架導覽時自動清理舊側欄並重新建構

6. **使用者設定（後台選項頁面）**
   - 隨時可調整、即時套用、跨裝置同步（透過 `chrome.storage.sync`）
   - 詳見下方「⚙️ 設定」段

## ⚙️ 設定

開啟設定頁面之方式：

- **方法一**：於網址列輸入 `chrome://extensions` → 於「司法院裁判書閱讀助手」卡片點選「詳細資料」→ 往下捲動至「擴充功能選項」並點開
- **方法二**：右鍵點擊 Chrome 右上角之擴充功能圖示 → 選「選項」

設定頁目前包含三個區塊：

| 區塊 | 設定項 | 說明 | 預設 |
|---|---|---|---|
| **複製設定** | 複製時自動附上裁判字號 | 開啟時，選取文字複製後尾端追加「（XX意旨參照）」；關閉時只移除分行不附字號。同一設定亦可在判決剪貼簿側邊欄頂端即時切換 | 開 |
| **參照耳標** | 顯示「參照」頁籤 | 開啟時於側邊欄新增「參照」頁籤，列出正文所有最高法院引用，並對括號附記長駐高亮；關閉後僅保留「判決架構」耳標 | 開 |
| **耳標位置** | 法令判解系統（FINT）左 / 右 | 「判決架構」直條耳標停靠在頁面左側或右側 | 左 |
| **耳標位置** | 裁判書系統（FJUD）左 / 右 | 同上，每個網站獨立設定 | 左 |
| **耳標位置** | 判解函釋 — 司法院內網（intraj_fint）左 / 右 | 同上，內網判解函釋獨立設定 | 左 |
| **耳標位置** | 裁判書系統 — 司法院內網（intraj_fjud）左 / 右 | 同上，內網裁判書獨立設定 | 左 |
| **展開深度** | 耳標展開到第幾層（1–6） | 1 = 只到 `主文/壹`、2 = `+一`、**3 = `+(一)/㈠`（預設）**、4 = `+1./⒈`、5 = `+(1)/⑴`、6 = `+①` | 3 |

所有設定**變更後即時套用**，已開啟之判決頁無須重新載入——側欄將於 1 秒內自動切換至新位置與新層級，複製功能亦即時遵循新規則。設定透過 `chrome.storage.sync` 儲存，登入同一 Google 帳號之其他 Chrome 裝置將自動同步。

## 🟡 由原始碼安裝（開發者）

若執行環境無法連接 Chrome Web Store，或擬自行修改程式、使用尚未發布之版本，可下載 GitHub 原始碼後，以 Chrome／Edge 之「開發人員模式」載入。

### 步驟一 — 下載原始碼 ZIP

1. 開啟 <https://github.com/han0302-cyber/judicial-outline-extension>
2. 點擊右上方綠色 **`Code`** 按鈕 → **`Download ZIP`**
3. 下載後解壓縮，取得 `judicial-outline-extension-main` 資料夾
4. 將資料夾搬移至**不致誤刪**之位置（例如 `~/Documents/` 或 `D:\Tools\`）

> ⚠️ **重要**：資料夾**不得再行移動、刪除或更名**；Chrome 將持續自該路徑載入擴充功能，一旦路徑變動即失效。

> 進階：若已安裝 git，亦可改以 `git clone https://github.com/han0302-cyber/judicial-outline-extension.git` 取得；後續更新僅需執行 `git pull`。

### 步驟二 — 於 Chrome／Edge 載入擴充功能

1. 於網址列輸入 `chrome://extensions`（Edge 為 `edge://extensions`）後按 Enter
2. 開啟右上角「開發人員模式」開關
3. 點選左上角「載入未封裝項目」
4. 選取先前解壓縮之 `judicial-outline-extension-main` 資料夾（內需包含 `manifest.json`）
5. 清單中出現「司法院裁判書閱讀助手」卡片，開關開啟後即完成

### 步驟三 — 測試

開啟任一判決頁（例如 <https://judgment.judicial.gov.tw/FJUD/default.aspx>），頁面左側應出現「判決架構」直條耳標；滑鼠移入即展開導覽卡片。

### 後續版本更新

- **ZIP 方式**：重新下載新版 ZIP，解壓後**覆蓋**原資料夾（路徑需一致），再於 `chrome://extensions` 卡片點「重新載入」🔄
- **git clone 方式**：執行 `git pull` 後，於 `chrome://extensions` 卡片點「重新載入」🔄

### 常見問題

- **`chrome://extensions` 未出現「載入未封裝項目」按鈕** → 右上角之「開發人員模式」開關尚未開啟
- **看不到耳標** → 確認當前頁面為「判決詳情」而非搜尋結果列表；列表頁不會注入側欄
- **重新啟動瀏覽器後擴充功能消失** → 表示原始碼資料夾已被移動或刪除，需重新「載入未封裝項目」指向正確路徑

## 🐞 疑難排解

- **看不到耳標** → 確認當前頁面為「判決詳情」而非搜尋結果列表；列表頁不會注入側欄
- **複製時未附上字號** → 確認選取之文字**位於判決正文區塊內**；選至頁首按鈕或導覽列將不會觸發。另請檢查設定頁或側邊欄頂端之「附上字號」是否關閉
- **欲複製但不納入剪貼簿卡片** → 改以 Cmd+X／Ctrl+X，剪貼簿內容照常正規化（含字號設定），惟不新增卡片
- **欲將耳標切換至右側** → 開啟設定頁（`chrome://extensions` → 詳細資料 → 擴充功能選項），於該網站選「右側」，無須重新載入即可生效
- **Safari 安裝後要求「允許未簽署之擴充功能」** → 當前安裝者為 Xcode 開發建置而非正式釋出之 ZIP；請由本檔之 GitHub Release 連結重新下載 ZIP，安裝後即無此需求
- **Chrome 開發人員模式載入之版本於重新啟動後消失** → 建議改由 Chrome Web Store 安裝，無須另行維護

## 🔐 技術細節

- 採 Manifest V3；核心為 content script 搭配輕量 service worker（僅設定側邊欄開啟行為與 `chrome.storage.session` 之存取層級）
- 判決剪貼簿側邊欄透過 Chrome `sidePanel` API 實作；資料存於 `chrome.storage.session`（記憶體，關閉瀏覽器即清空），字體大小偏好存於 `chrome.storage.local`
- 以 `TreeWalker` 扁平化 `#jud`／`#plCJData` 等正文容器之文字節點，並維護每個文字節點於扁平字串中之偏移對照表
- 於扁平字串上執行三回合偵測（物理行首、句末行內、CJK 圓圈數字）後，以 `splitText` 於對應文字節點插入隱形 `<span id>` 錨點，不影響原排版
- 側欄 DOM 掛載於 `window.top.document`，使 `position: fixed` 以最外層視窗為基準（解決 FJUD 將內容置入內嵌框架時，fixed 行為退化為 absolute 之問題）
- 點擊跳轉時沿內嵌框架鏈計算目標於最外層視窗之絕對 Y 座標，單次 `scrollTo` 並扣除固定表頭高度，避免 `scrollIntoView` 與延遲 `scrollBy` 之動畫競爭
- 「前往」功能以書籤錨點為主軸：複製當下於選取範圍起迄各插入空 `<span id>`，其 id 一併寫入卡片；側邊欄以 `chrome.tabs.sendMessage` 對所有司法院網域之分頁廣播 `hasAnchor`，命中之分頁即為原頁面。切換過去後以 `Range.setStartAfter`／`setEndBefore` 建立區段並透過 CSS Highlight API 上色；錨點不在時退回 `pageUrl`／`sourceUrl` 比對與 `rawText` 精確文字搜尋兩層備援
- 引文抽取採「全文掃描＋鄰近分群」：先建立去除空白之搜尋字串並維護其與原始扁平字串之索引對照表（以吸收 FJUD／FINT 之軟斷行），於搜尋字串上以正規表示式比對八類案號，再依鄰近度合併為引文條目；對行內式引用嚴格要求尾端強結論語（意旨參照／可參／可資參照等），以避免誤收法院正在排斥之先例
- Safari 版本以 `xcrun safari-web-extension-converter` 自同一份 Chrome 原始碼產生 Xcode 專案，編譯為 Safari Web Extension
